# Data Model

Schema source: `server/db/schema.sql`.
Runtime migrations: `server/src/db.js` `MIGRATIONS` array.

## Core Tables

| Table | Purpose |
|---|---|
| `users` | Login, role, team scope |
| `teams` | Requesting teams/departments |
| `suppliers` | Vendors and payment/delivery metadata |
| `categories` | Product/category dictionary |
| `products` | SKU catalog |
| `orders` | Procurement order headers |
| `order_items` | Order line items |
| `order_suppliers` | Per-order supplier commercial terms and custom metadata |
| `purchase_requests` | Request headers before order |
| `request_items` | Request line items |
| `contracts` | Contract/PO/DDH records and generated document HTML |
| `email_logs` | Outbound email history |
| `ratings` | Supplier/requester survey ratings |
| `warehouse_stock` | Current stock snapshot |
| `inventory_moves` | Source of truth for stock movements |
| `settings` | Key-value app config |
| `attachments` | DB-backed images/files |
| `workflow_states` | Configurable order statuses |
| `order_status_history` | Status audit history |
| `notifications` | In-app notifications and action tasks |
| `signatories` | Default signers by role/scope |
| `backup_*` | MySQL-to-Google-Sheets backup config/state/log/lock |

## Important Relationships

- `orders.team_id -> teams.id`
- `orders.supplier_id -> suppliers.id`
- `order_items.order_id -> orders.id`
- `order_suppliers.order_id -> orders.id`; `order_suppliers.supplier_id -> suppliers.id`
- `order_items.product_id -> products.id`
- `order_items.category_id -> categories.id`
- `purchase_requests.team_id -> teams.id`
- `request_items.request_id -> purchase_requests.id`
- `contracts.order_id -> orders.id`
- `contracts.supplier_id -> suppliers.id`
- `users.team_id -> teams.id`

## Orders

Important header fields:

- `order_code`: business code, generated by `nextOrderCode`
- `requester_email`, `requester_name`
- `team_id`, `supplier_id`
- `project_name`, `pm`, `hang_muc`
- `status`, `status_raw`
- `request_date`, `expected_date`, `actual_date`, `handover_date`
- `receiving_point`
- `pr_no`, `contract_no`, `po_no`
- `payment_method`, `payment_term`
- `qdnb_link` (optional internal-decision URL)
- `total_amount`
- `warehouse_status`
- `custom_fields`
- `deleted_at`, `deleted_by`

`orders.status` is `VARCHAR(50)` after migration to support configurable workflow.

## Order Items

Important fields:

- item/category/product references
- `item_name`, `item_code`, `description`, `unit`
- `quantity`, `unit_price`, `vat_rate`, `discount_rate`
- `thanh_tien`, `tien_thue`, `line_total`
- `quotation_url`, `design_link`, `reason_choose`
- `supplier_id`, `master_contract`, `so_pr`
- `progress`, `nhap_kho`, `in_catalog`
- rental fields

Line totals are calculated in `server/src/routes/orders.js` `computeLine`.

Supplier payment/contract/vendor values that vary by order belong in `order_suppliers`, not `suppliers`. `custom_fields` keeps this relationship extensible.

## Warehouse

`inventory_moves` is the ledger. `warehouse_stock` is rebuilt from moves.

Voucher types:

- `PNK`: inbound
- `PXK`: outbound

`rebuildStock` in `server/src/routes/warehouse.js` recalculates current stock.

## Contracts

`contracts.document_html` stores generated HTML. DOCX download renders from uploaded/default DOCX templates.

Auto-create threshold is 20,000,000 VND.

Contract type decision:

- `DDH` if supplier has valid master contract
- `HD` otherwise

## Settings Keys

Common keys:

- `ui_theme`
- `ui_labels`
- `company_info`
- `smtp_config`
- `tpl_po`
- `tpl_hd`
- backup keys for Google Sheets

`settings.value` is `LONGTEXT` because template base64 can be large.

## Migration Rule

When adding columns:

1. Update `server/db/schema.sql` for fresh DBs.
2. Add an idempotent `ALTER TABLE` entry to `MIGRATIONS` in `server/src/db.js`.
3. Handle duplicate-column errors only as currently done.
4. Do not make migrations destructive unless the user explicitly asks.
